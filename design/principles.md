# LazyDesign Principles

LazyDesign combines Material 3 system thinking, Monet dynamic color, Vercel restraint, Linear productivity, HeroUI polish, shadcn composition, and GSAP-grade motion orchestration. The combination is unified by these principles.

## 1. Flat First

The default interface is flat. Hierarchy comes from role, contrast, density, border, state, and rhythm.

## 2. Density With Dignity

LazyDesign supports serious SaaS density without sacrificing reading, targeting, focus, or trust.

## 3. Design Over Decoration

The system earns quality through typography, spacing, color roles, and interaction. It does not rely on gradient, shadow, glass, or glow as a visual shortcut.

## 4. Dynamic, Not Random

Monet color is a theme engine, not a decoration engine. Dynamic color must resolve into stable semantic roles and accessible state layers.

## 5. Motion With Purpose

Motion exists to explain change. It should be short, interruptible, reversible, and reduced-motion aware.

## 6. Runtime Before Components

Tokens, resolver, registry, and CSS variables come before components. A component that cannot survive theme switching is not production ready.

## 7. Ownership Over Black Boxes

LazyDesign follows the shadcn idea that teams should be able to read, compose, and own the component code they ship.

## 8. Accessibility By Default

Contrast, focus, semantics, keyboard behavior, reduced motion, and screen-reader compatibility are not optional enhancements.
